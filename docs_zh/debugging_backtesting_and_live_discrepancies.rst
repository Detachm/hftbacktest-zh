==========================
排查回测与实盘差异
==========================

把实盘值和回测值画在同一张图上，是一个很好的起点。强烈建议至少比较权益曲线和持仓曲线。此外，可视化 alpha、订单价格等信息，也有助于定位差异来源。

[Image]

如果回测策略已经正确实现到实盘中，观察到的差异通常主要来自两个因素。

1. 延迟

延迟包括行情延迟和订单延迟，它对准确回测非常关键。强烈建议自行采集数据，以准确测量你这一侧的行情延迟。如果使用外部数据源，也需要确认其中的行情延迟与你的实际延迟一致。

订单延迟应从你这一侧测量。可以通过记录订单行为，或定期提交远离 mid-price、不会成交的订单并随后撤单，来测量并记录订单延迟。

也可以人为降低延迟，用来评估低延迟对策略表现的改善。这有助于评估更高级别的交易所项目、流动性提供者项目，以及基础设施投入带来的影响。理解更好的基础设施是否带来竞争优势是有价值的。

2. 队列模型

选择能准确反映实盘结果的队列模型非常重要。你可以开发自己的队列模型，也可以使用已有模型。HftBacktest 提供了几个主要队列模型，例如 ``PowerProbQueueModel`` 系列，可通过调整模型参数来贴近实盘结果。更多信息见 :ref:`ProbQueueModel <order_fill_prob_queue_model>`。

需要特别注意的是，回测是在“无市场冲击”的假设下进行的。市价单，或会吃掉流动性的限价单，可能产生市场冲击，从而造成回测和实盘差异，也会让成交模拟变得困难。此外，如果限价单数量过大，部分成交及其市场冲击也可能带来差异。建议先用小规模交易对齐回测和实盘结果，再逐步增加交易规模，并持续观察两者差异。
